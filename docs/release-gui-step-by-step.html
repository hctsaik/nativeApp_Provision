CIM 平台版本發佈 使用說明書

這是在做什麼？ 把平台打包成一個「正式版資料夾」，拿去現場機器安裝。

我需要什麼、要多久？ 一台已設定好的發佈電腦（見下方「開始前你需要什麼」）， 全程約 5 分鐘。不需要記密碼、不需要懂任何技術名詞。

怎麼知道成功？ 最後會得到一個 production-版次 的資料夾＋一段交付指示； 複製到 USB 交給現場，就完成了。

開始前你需要什麼（一次確認就好）

  1. 這台電腦要有平台原始碼 C:\code\claude\nativeApp（工具會自動偵測到）。
  2. apps\host-tauri\prebuilt\cim-light.exe 已就位（缺了工具會擋下並提示，交給工程師處理）。
  3. 命令列打 py -3.11 --version 有反應（Python 3.11）。
  4. 金鑰（防偽用的鑰匙檔）不用先建——第一次發版會自動建立，你不用做任何事。
「命令列」是什麼、怎麼打？（30 秒教學）
  1. 按鍵盤左下角的 Windows 鍵，直接輸入 cmd，按 Enter——會開出一個黑色視窗。
  2. 在黑窗裡輸入 py -3.11 --version，按 Enter。
  3. 「有反應」＝顯示 Python 3.11.x；顯示紅字或「不是內部或外部命令」＝沒有，找工程師裝。

先認識 6 個畫面上會出現的詞（看過就好，不用背）

畫面上的詞意思

現場安裝與更新（單頁）

你手上應該有：一個 production-版次 資料夾（USB 或共享資料夾拿到的）。

你需要什麼

  1. 這台機器已裝 Python 3.11（命令列打 py -3.11 --version 有反應）， 或已由 IT 把可攜 Python 放在 %LOCALAPPDATA%\CIM-Platform\runtime\python311\。 安裝包資料夾不自帶 Python。
    （怎麼確認：按 Windows 鍵→輸入 cmd→Enter，在黑窗輸入 py -3.11 --version→Enter；顯示 Python 3.11.x 就是有。）
  2. Windows 10/11（需要 Microsoft Edge WebView2，一般內建）。

安裝（第一次）

  1. 把整個 production-版次 資料夾複製到這台機器的任何位置（例如 D:\CIM 或桌面）。
  2. 雙擊資料夾裡的 install.bat，等黑窗跑完。
  3. 啟動平台：%LOCALAPPDATA%\CIM-Platform\bin\start-platform.bat （可傳送捷徑到桌面，以後都從這裡開）。
【對】黑窗印出「結果:UPDATED」＝安裝成功 （第一次安裝就是顯示 UPDATED，這是正常的）。

之後每次更新

拿到新的 production 資料夾 → 雙擊新資料夾裡的 install.bat。就這樣。 （機器上的資料與設定不會動；來路不明的更新包會被自動拒絕。）

出狀況怎麼辦

你看到什麼怎麼辦
雙擊 install.bat 後出現 FAILED、untrusted 或「缺 offline-channel」 這包東西不對或沒複製完整。停手，把整個資料夾重新複製一次再雙擊； 還是失敗→找發佈的人重新拿一份。被 SmartScreen 攔→按「其他資訊→仍要執行」； 被公司電腦管制攔→交給 IT。
裝完了，雙擊 start-platform.bat 沒畫面或一閃就關 先確認 py -3.11 --version 有反應（沒有→回到上面「你需要什麼」）； 有 Python 仍沒畫面→把 %LOCALAPPDATA%\CIM-Platform 底下的 applications 資料夾裡的 logs 壓縮起來，交給發佈的人。
