0. 這件事的本質

CIM 平台的 Tauri 殼(cim-light.exe)不能被本機重新編譯,也不能被叫去開啟任意 URL。 它在啟動時會拉起 Python engine,engine 掃描 plugin.yaml 建出「工具清單」, 殼把清單畫進 portal 的「工作流程」下拉,User 按 Start,engine 才把那個工具的 Streamlit 起在一個空閒埠上,並把 URL 交給殼去 iframe。

所以「把 CV Viewer 接進殼」不是改殼,而是:讓 engine 認得 CV Viewer 這個工具。 你只要放兩個檔案(一份 plugin.yaml、一支 runner),engine 下次開機就會自己掃到它。

2

照著 app-lv 抄:它跟 CV Viewer 是同一種形狀

平台裡已經有一個活生生的範本:LV (VisualLatent)。它跟 CV Viewer 一樣是 扁平的 scripts/ 式 App —— app.py 直接 import 同層的兄弟模組, 而不是一個可安裝的套件。這正是 CV Viewer 的形狀(5_PG_Develop/app.py import casepkg…)。

先讀這兩個現成檔案,它們是你要仿造的樣板:

檔案它做什麼
sidecar/python-engine/plugins/lv/modules/app-lv/plugin.yaml 宣告 id: app-lv / runner: lv / category: app / requires:(per-tool venv 的相依)
sidecar/python-engine/tools/lv_runner.py 把 LV 的 scripts/ 塞進 sys.path,再用 runpy.run_path(..., run_name="__main__") 執行 app.py —— 等同於 streamlit run scripts/app.py

為什麼要 runner,而不是直接指向 app.py? 已驗證

因為扁平式 App 的 import casepkg 只有在「app.py 自己的目錄在 sys.path 上」時才成立。 streamlit run x/y/app.py 會把 腳本自己的目錄 放進 sys.path —— runner 就是在複製這個行為。 lv_runner.py 的 docstring 把這件事寫得很清楚:「LV is a flat scripts/ app … so this runner puts LV's scripts/ dir on sys.path and runpy-executes app.py as __main__」。

把 CV Viewer 的原始碼接進來

LV 與 AI4BI 是用 git submodule 掛在 sidecar/python-engine/vendor/ 底下的 (.gitmodules 已驗證)。CV Viewer 目前沒有 GitHub remote, 所以開發階段有兩條路,選一條,不要兩條都做:

路線 A(推薦,開發階段):用環境變數指到本機 repo

不動 CV Viewer 的 repo、不需要 GitHub。runner 讀一個環境變數,預設值指向 vendor,找不到就退回本機路徑。

路線 B(要進版控時):加成 submodule

85 MB 的權重檔不要進 submodule。LV 就是這樣處理的 —— 模型權重不入 submodule, 由 LV_MODELS_DIR 指向平台的 model-house。CV Viewer 照抄:用 CVR_MODELS_DIR 指過去。

4

建立 runner:tools/cv_viewer_runner.py

路徑必須剛好是這個 —— 因為 runner: cv_viewer 會被 engine 轉成 cv_viewer_runner.py(engine.py:689)。

# C:\code\claude\nativeApp\sidecar\python-engine\tools\cv_viewer_runner.py
"""Streamlit runner for the CV Viewer embedded app (``runner: cv_viewer`` → this file).

CV Viewer is a *flat* app, exactly like LV: ``5_PG_Develop/app.py`` imports its
sibling modules (``import casepkg`` / ``cocoio`` / ``viewer`` …, 23 of them) rather
than being an installable package. `streamlit run x/app.py` puts the SCRIPT'S OWN
directory on sys.path, and that is the only reason those imports resolve — so this
runner reproduces exactly that, then runpy-executes app.py as __main__.

Model weights (85 MB dinov2_vits14.pth) are NOT vendored: the platform points at a
writable model-house via CVR_MODELS_DIR, mirroring LV_MODELS_DIR.
"""
from __future__ import annotations

import os
import runpy
import sys
from pathlib import Path

_ENGINE_ROOT = Path(__file__).resolve().parent.parent

# vendor/ 優先(進了版控之後就是這裡);開發階段可用 CV_VIEWER_ROOT 指到本機 repo。
_DEFAULT = _ENGINE_ROOT / "vendor" / "CV_Viewer"
_CV_ROOT = Path(os.environ.get("CV_VIEWER_ROOT") or _DEFAULT)
_CV_APP_DIR = _CV_ROOT / "5_PG_Develop"

if not (_CV_APP_DIR / "app.py").is_file():
    raise SystemExit(
        f"[cv_viewer_runner] 找不到 CV Viewer 的入口:{_CV_APP_DIR / 'app.py'}\n"
        f"  開發階段請設環境變數 CV_VIEWER_ROOT 指到 CV_Viewer repo,例如:\n"
        f"    set CV_VIEWER_ROOT=C:\\code\\claude\\CV_Viewer\n"
        f"  或把 CV_Viewer 掛成 submodule 到 {_DEFAULT}"
    )

# 權重:CV Viewer 自己就支援 CVR_MODELS_DIR —— app.py:1050 的註解直接寫著
# 「CVR_MODELS_DIR 讓 Tauri 殼日後把它的共用模型夾接進來,不需要改任何程式」。
# 不設也能跑(app.py 用 Path(__file__).parents[1]/'models',不依賴 cwd),
# 設了就能把權重放到平台的共用 model-house,不必塞進每個交付包。
os.environ.setdefault("CVR_MODELS_DIR", str(_CV_ROOT / "models"))

# 這一行就是整個 runner 的理由:扁平式 App 的 sibling import 只有這樣才成立。
if str(_CV_APP_DIR) not in sys.path:
    sys.path.insert(0, str(_CV_APP_DIR))

runpy.run_path(str(_CV_APP_DIR / "app.py"), run_name="__main__")

AI 注意:sys.path.insert 只影響「這個 Streamlit 子程序」。 engine 對每一個 app 工具會各開一個程序(engine.py:1411 _start_app), 所以 CV Viewer 的通用模組名(viewer / models)不會污染別的外掛。 這是設計,不是巧合 —— lv_runner.py 的註解特別講了這件事。

啟動 CV Viewer 之後,原本在跑的工具不見了 這是設計:同時只能有一個 active 工具 engine.py:1018-1020 — start() 第一件事就是 self.stop()。portal 也會在有工具執行時把下拉 disable 掉。

收尾:別留孤兒程序。engine 是 python.exe(不是 engine.exe), Streamlit 又是 engine 的子程序,而 engine 崩潰會自動換埠重啟(3 秒後,sidecar.rs:198-241)—— 所以「對記錄下來的埠送 shutdown」會漏掉新埠。

正常關窗是安全的:殼會攔 CloseRequested → POST /shutdown → 等 5 秒 → 才 kill (lib.rs:79-89、sidecar.rs:244-285)。強制砍 exe 才會留孤兒。

要清殘留,用 start-dev-tauri.bat 自己用的那一招(只殺屬於這個平台的,不會誤殺使用者其他的 python):

powershell -NoProfile -Command "Get-CimInstance Win32_Process | Where-Object { $_.Name -eq 'python.exe' -and $_.CommandLine -like '*engine.py*--control-port*' } | ForEach-Object { Stop-Process -Id $_.ProcessId -Force }"
